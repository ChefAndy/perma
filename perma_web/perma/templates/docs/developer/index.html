API Overview

The Perma.cc API helps you create and manage archives without using your web browser.

There are a number of endpoints, but you really only manipulate two "things," archives and folders.

The API is RESTful and leverages the HTTP GET, POST, PUT, PATCH, and DELETE verbs.

Please let us know if you have suggestions or run into bugs in the API.

API Key

Many of the API endpoints will require you to use an API key. Find your API key by clicking here (or navigating to the Tools section, which can be found in your Dashboard under the Settings section). Click on the Generate an API key button.

Pagination

By default, the API will return results in "pages" of up to 300 items. If the response comprises more than 300 results, you can page through the items using the offset parameter. For example, if you want to start at the 301st result, set the offset value to 1 (we start counting at 0, of course).

curl https://api.{{ request.get_host }}/v1/public/archives/?offset=1

If you'd prefer a smaller or larger result set, specify the desired size of the pages using the limit parameter. In this example, we request archives returned one at a time.

curl https://api.{{ request.get_host }}/v1/public/archives/?limit=1

Examples in cURL

Our API documentation is example based, and our examples show you how to interact with the API using the cURL command line utility. A friendly tip - you can add the -v option to help debug your HTTP interactions.

curl https://api.{{ request.get_host }}/v1/public/archives/?limit=1 -v

Public Archives

The Public Archives API endpoint helps you find all publicly available Perma.cc archives. You do not need an API Key to use this portion of the API.

The base endpoint for this API is https://api.{{ request.get_host }}/v1/public/archives/

Get all public archives

Use HTTP GET to retrieve every public archive in Perma.cc. (In order to keep this example short, we limit the number of returned items to one.)

curl https://api.{{ request.get_host }}/v1/public/archives/?limit=1

Response:

{"meta":{"limit":1,"next":"https://api.{{ request.get_host }}/v1/public/archives/?limit=1&offset=1","offset":0,"previous":null,"total_count":99999},"objects":[{"guid":"W5MF-N9EV","creation_timestamp":"2018-05-14T15:12:33Z","url":"http://example.com","title":"Example Domain","description":null,"warc_size":19156,"warc_download_url":"http://{{ request.get_host }}/v1/public/archives/W5MF-N9EV/download","captures":[{"role":"primary","status":"success","url":"http://example.com","record_type":"response","content_type":"text/html","user_upload":false,"playback_url":"//{{ request.get_host }}/warc/W5MF-N9EV/http://example.com"},{"role":"screenshot","status":"success","url":"file:///W5MF-N9EV/cap.png","record_type":"resource","content_type":"image/png","user_upload":false,"playback_url":"//{{ request.get_host }}/warc/W5MF-N9EV/id_/file:///W5MF-N9EV/cap.png"}],"queue_time":3,"capture_time":4}]}

curl https://api.{{ request.get_host }}/v1/public/archives/W5MF-N9EV/

Download a single public archive

Perma archives are downloadable and can be viewed using tools that can replay WARC files, like Rhizome's Webrecorder Player.

wget https://api.{{ request.get_host }}/v1/public/archives/Y6JJ-TDUJ/download

or

curl -o your_favorite_filename.warc.gz https://api.{{ request.get_host }}/v1/public/archives/Y6JJ-TDUJ/download

User

The User API endpoint returns basic information about your account.

The base endpoint for this API is https://api.{{ request.get_host }}/v1/user/

Get your account details

Use GET to retrieve your account details. (You only have access to your own account.)

curl https://api.{{ request.get_host }}/v1/user/?api_key=your-api-key

The response includes your user ID and a list of all the top-level folders you have access to. For more information about these folders, their subfolders, and their associated organizations, use the Folders and Organizations API endpoints.

{"id": 4, "first_name": "Jane", "last_name": "Doe", "full_name": "Jane Doe", "short_name": "Jane", "top_level_folders":[{"id":25,"name":"Personal Links","parent":null,"has_children":true,"organization":null}]}
{% include "docs/developer/toc.html" with section="organizations" %}

Organizations

The Organizations API returns basic information about the organizations you belong to or manage.

The base resource for this API is https://api.{{ request.get_host }}/v1/organizations/

Get your organizations

Use GET to retrieve a list of all of the organizations you belong to or manage.

curl https://api.{{ request.get_host }}/v1/organizations/?limit=1&api_key=your-api-key

Response.

{"meta": {"limit": 20, "next": null, "offset": 0, "previous": null, "total_count": 1}, "objects": [{"id": 1, "name": "Test Journal"}]}

Get an organization's details

Use GET to retrieve details about an organization.

curl https://api.{{ request.get_host }}/v1/organization/1?api_key=your-api-key

The response includes the ID of the organization's "shared folder", the folder in which all archives belonging to the organization are stored. For more information about this folder and its subfolders, as well as more information about the archives stored in those folders, use the Folders API endpoint.

Archives

      
The Archives API helps you create and manage archives.

      
The base resource for this API is https://api.{{ request.get_host }}/v1/archives/


      
Create an archive

      
Use POST to create a new archive.

      
Include the URL as JSON-encoded data.

      
curl -H 'Content-Type: application/json' -X POST -d '{"url": "http://example.com"}' https://api.{{ request.get_host }}/v1/archives/?api_key=your-api-key

      
Optional: to create an archive in a specific folder, include the folder's ID number in the JSON-encoded data. This should be a single integer, even for nested folders. The example below creates an archive in folder 1, and then another in folder 27 (a subfolder of folder 1). If you don't specify a folder ID, your archive will be created in your "Personal Links" folder.

      
Optional: you may provide a short "title" for the archive in the JSON-encoded data. If you do not specify a title, Perma will attempt to retrieve a title from the contents of the archived URL.


curl -H 'Content-Type: application/json' -X POST -d '{"url": "http://example.com", "title": "This is an example site", "folder": 1}' https://api.{{ request.get_host }}/v1/archives/?api_key=your-api-key
curl -H 'Content-Type: application/json' -X POST -d '{"url": "http://example.com", "title": "This is another example site", "folder": 27}' https://api.{{ request.get_host }}/v1/archives/?api_key=your-api-key

      
The response includes detailed information about the newly created archive:

      
{"assets": [{"base_storage_path": "2015/1/21/21/41/8H4S-TNRA", "favicon": null, "image_capture": "pending", "pdf_capture": null, "warc_capture": "pending"}], "created_by": {"first_name": "Regular", "full_name": "Regular User", "id": 4, "last_name": "User", "short_name": "Regular"}, "creation_timestamp": "2015-01-21T16:41:15Z", "is_private": false, "expiration_date": "2017-01-21T16:41:15Z", "folders": ["/v1/folders/25/"], "guid": "8H4S-TNRA", "notes": "", "title": "This is a test page", "url": "http://example.com", "organization": null}


      
View the details of one archive

      
If you have the globally unique ID (GUID) of an archive, use GET to retrieve its details. More details are available via this authenticated Archives endpoint than are available from the non-authenticated Public Archives endpoint.

      
curl https://api.{{ request.get_host }}/v1/archives/Y6JJ-TDUJ/?api_key=your-api-key

      
Response:

      
{"assets": [{"base_storage_path": "2015/1/21/22/39/Y6JJ-TDUJ", "favicon": null, "image_capture": "cap.png", "pdf_capture": null, "warc_capture": "archive.warc.gz"}], "created_by": {"first_name": "Regular", "full_name": "Regular User", "id": 4, "last_name": "User", "short_name": "Regular"}, "creation_timestamp": "2015-01-21T17:39:44Z", "is_private": false, "expiration_date": "2017-01-21T17:39:44Z", "folders": ["/v1/folders/25/"], "guid": "Y6JJ-TDUJ", "notes": "", "title": "Example Domain", "url": "http://example.com", "organization": null}


      
Download a single archive

      
Perma archives are downloadable and can be viewed using tools that can replay WARC files, like Rhizome's Webrecorder Player.

      
wget https://api.{{ request.get_host }}/v1/archives/Y6JJ-TDUJ/download?api_key=your-api-key

Make an archive private

      
Use PATCH to make an archive private.

      
Include the GUID of the archive in the URL, and set the archive's "is_private" field to true using JSON-encoded data: 

      
curl -H 'Content-Type: application/json' -X PATCH -d '{"is_private": true}' https://api.{{ request.get_host }}/v1/archives/7J9R-2QM5/?api_key=your-api-key

      
Response:

      
{"assets": [{"base_storage_path": "2015/1/21/22/39/Y6JJ-TDUJ", "favicon": null, "image_capture": "cap.png", "pdf_capture": null, "warc_capture": "archive.warc.gz"}], "created_by": {"first_name": "Regular", "full_name": "Regular User", "id": 4, "last_name": "User", "short_name": "Regular"}, "creation_timestamp": "2015-01-21T17:39:44Z", "is_private": true, "expiration_date": "2017-01-21T17:39:44Z", "folders": ["/v1/folders/25/", "/v1/folders/27/"], "guid": "Y6JJ-TDUJ", "notes": "", "title": "Example.com. yo.", "url": "http://example.com", "organization": {"id": 1, "name": "Test Journal"}}

Use PATCH to change an archive's notes or title field.

      
Include the GUID of the archive in the URL, and specify your desired changes using JSON-encoded data: 

      
curl -H 'Content-Type: application/json' -X PATCH -d '{"title": "My updated title"}' https://api.{{ request.get_host }}/v1/archives/Y6JJ-TDUJ/?api_key=your-api-key 

      
Response:

Move an archive

      
Use PUT to move an archive into a different folder.

      
Include the ID of the destination folder as the first variable in the URL and the GUID of the archive as the second. The below example moves Y6JJ-TDUJ into folder 33.

      
curl -X PUT https://api.{{ request.get_host }}/v1/folders/33/archives/Y6JJ-TDUJ/?api_key=your-api-key -v

      
Response:

      
{"assets": [{"base_storage_path": "2015/1/21/22/39/Y6JJ-TDUJ", "favicon": null, "image_capture": "cap.png", "pdf_capture": null, "warc_capture": "archive.warc.gz"}], "created_by": {"first_name": "Regular", "full_name": "Regular User", "id": 4, "last_name": "User", "short_name": "Regular"}, "creation_timestamp": "2015-01-21T17:39:44Z", "is_private": true, "expiration_date": "2017-01-21T17:39:44Z", "folder": "33", "folders": ["/v1/folders/25/", "/v1/folders/33/"], "guid": "Y6JJ-TDUJ", "notes": "", "title": "Example.com. yo.", "url": "http://example.com", "organization": {"id": 1, "name": "Test Journal"}}


      
Delete an archive

      
Use DELETE to delete an archive.

      
curl -X DELETE https://api.{{ request.get_host }}/v1/archives/7J9R-2QM5/?api_key=your-api-key

      
This request deleted the archive. There will be no content in the HTTP response. You will see the a 204 HTTP status code returned.

      
No response data. HTTP status will be 204.

Folders


      
The Folders API helps you create and manage folders.

      
The base resource for this API is https://api.{{ request.get_host }}/v1/folders/


      
Create a folder

      
Use POST to create a new folder.

      
New folders can only be created as "children" (subfolders) of existing folders. Include the ID of the "parent" folder in the URL. The below example creates a new folder inside folder 25.

      
curl -H 'Content-Type: application/json' -X POST -d '{"name": "a new folder"}' https://api.{{ request.get_host }}/v1/folders/25/folders/?api_key=your-api-key

      
The response will include the ID of the new folder.

      
{"id": 35, "name": "a new folder", "parent": "/v1/folders/25/","has_children": false, "organization": null}


      
View a folder's details

      
Use GET to retrieve details about an existing folder.

      
curl -v "https://api.{{ request.get_host }}/v1/folders/35/?api_key=your-api-key"

      
{"id": 35, "name": "a new folder", "parent": "/v1/folders/25/","has_children": false, "organization": null}


      
View a folder's subfolders

      
Use GET to list a "parent" folder's "children" (subfolders). Include the ID of the parent folder in the URL.

      
curl -v "https://api.{{ request.get_host }}/v1/folders/25/folders?api_key=your-api-key"

      
{"id": 35, "name": "a new folder", "parent": "/v1/folders/25/","has_children": false, "organization": null}


      
View a folder's archives

      
Use GET to list the archives stored in a folder.

      
curl -v "https://api.{{ request.get_host }}/v1/folders/25/archives?api_key=your-api-key"

      

      
Note that archives stored in the folder's subfolders are not included.


      
Rename a folder

      
Use PATCH to change a folder's name.

      
curl -H 'Content-Type: application/json' -X PATCH -d '{"name": "a new folder name"}' 'https://api.{{ request.get_host }}/v1/folders/35/?api_key=your-api-key'

      
Response:

      
{"id": 35, "name": "a new folder name", "parent": "/v1/folders/25/","has_children": false, "organization": null}


      
Move a folder

      
Use PUT to move a folder into another folder.

      
Include the ID of the destination folder as the first variable in the URL, and include the ID of the folder you want to move as the second variable. For example, the below moves folder 36 into folder 35.

      
curl -H 'Content-Type: application/json' -X PUT https://api.{{ request.get_host }}/v1/folders/35/folders/36/?api_key=your-api-key

      
Response:

      
{"id": 36, "name": "just another folder", "parent": "/v1/folders/35/", "has_children": false, "organization": null}


      
Delete a folder

      
Use DELETE to delete a folder.

      
The below example deletes folder 36.

      
curl -X DELETE https://api.{{ request.get_host }}/v1/folders/36/?api_key=your-api-key

      
There will be no content in the HTTP response. You will see the a 204 HTTP status code returned.

      
No response data. HTTP status will be 204.


      
View top-level folders

      
Use GET to retrieve a list of all the top-level folders you have access to. This information is also available at the User API endpoint.

      
curl -X DELETE https://api.{{ request.get_host }}/v1/folders?api_key=your-api-key

      
Response:

      
No response data. HTTP status will be 204.

      
Use additional Folders API endpoints to retrieve details about these folders, including listing their subfolders and the archives they contain.

Capture Jobs


      
The Capture Jobs API exposes details about the process of creating an archive, internally referred to as a "Capture Job".

      
The base resource for this API is https://api.{{ request.get_host }}/v1/capture_jobs/


      
Get your ongoing capture jobs

      
List all your pending or in_progress capture jobs.

      
curl -v https://api.{{ request.get_host }}/v1/capture_jobs/?api_key=your-api-key


      
Get a single archive's capture status

      
View capture status for a particular archive by GUID.

      
curl -v https://api.{{ request.get_host }}/v1/capture_jobs/ABCD-1234/?api_key=your-api-key
